# 交接說明：換到其他 AI / 工具時怎麼繼續

這份文件說明如何把本專案交接給「其他 AI 助手」或「其他編輯器 / 開發工具」繼續開發。

## 一句話重點

本專案是**純前端的標準 HTML / CSS / JavaScript**，沒有後端、沒有框架、沒有安裝依賴，因此**不綁定任何特定 AI 或工具**，任何人、任何編輯器、任何 AI 都能接手。

## 專案組成

```
圖片壓縮網站/
├─ index.html     # 頁面結構
├─ styles.css     # 視覺樣式（Apple 風格）
├─ app.js         # 全部功能邏輯（壓縮、預覽、對比、ZIP 打包）
├─ README.md      # 使用說明
└─ HANDOFF.md     # 本檔
```

沒有 `node_modules`、沒有建置步驟（build step）、沒有環境變數。檔案本身就是可執行的成品。

## 怎麼執行 / 預覽

- 最簡單：**雙擊 `index.html`** 用瀏覽器開啟。
- 或在資料夾內啟動小型伺服器：`python -m http.server 8123`，再開 <http://localhost:8123>。

## 交接給「其他 AI 助手」

### A. 整合型工具（如 Cursor、Windsurf、VS Code + Copilot 等）
直接用該工具開啟這個資料夾即可，AI 能讀寫檔案、執行指令。把下面這段貼給它當開場白即可：

> 這是一個純前端圖片壓縮網站，只有 index.html、styles.css、app.js 三個程式檔，無後端、無框架、無依賴。所有壓縮在瀏覽器以 Canvas API 完成。請先讀過這三個檔案再進行修改。

### B. 對話型 AI（如 claude.ai、ChatGPT 網頁版、Gemini 等）
這類沒有直接改檔案的能力，流程是「複製貼上」：
1. 把 `index.html`、`styles.css`、`app.js` 三個檔的內容貼給它。
2. 說明你要改什麼。
3. 把它給的修改結果**貼回**對應檔案、存檔。

> 提示：一次只貼相關檔案、明確說「只改 X 功能、其他別動」，可減少它擅自重寫整份程式。

## 交接給「其他編輯器 / 開發者」

把整個資料夾（或 `圖片壓縮網站.zip`）給對方即可，不需任何前置安裝。對方只要有現代瀏覽器就能執行與修改。

## 修改時的關鍵須知

給接手的人 / AI 的重點提醒：

- **全部邏輯集中在 `app.js`**，主要區塊：載入與壓縮、目標大小搜尋、ZIP 打包、滑動對比、事件綁定。
- 壓縮以 **Canvas `toBlob`** 完成；PNG 為無損格式，品質參數無效。
- **JPEG 不支援透明**，輸出前會先填白底（避免透明區變黑）。
- **ZIP 為自行實作**的 store（不壓縮）寫入器，位於 `buildZip()`；無第三方函式庫。
- 拖動滑桿時用 debounce + 版本 token，確保只套用最後一次壓縮結果。

## 部署（給公開網址）

無後端，可直接丟到任一靜態託管，免費且自帶 HTTPS：

- **GitHub Pages**、**Netlify**、**Vercel**、**Cloudflare Pages**

最快的方式（GitHub Pages）：
1. 把專案推到 GitHub repo。
2. Repo → Settings → Pages → Source 選 `main` 分支根目錄。
3. 稍候即可取得公開網址。
